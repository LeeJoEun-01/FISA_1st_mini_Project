# 📝 Git Commit 메시지 규칙

- **형식**
  
  ```
  <타입>: <메시지>
  ```

- **타입 예시**
  
  - `feat`: 새로운 기능
  
  - `fix`: 버그 수정
  
  - `docs`: 문서 변경
  
  - `style`: 스타일(포맷) 변경
  
  - `refactor`: 리팩토링
  
  - `test`: 테스트 추가/수정
  
  - `chore`: 기타 작업

- **메시지 규칙**
  
  - 현재 시제(동사) 사용 (예: `add`, `fix`, `update`)
  
  - 첫 글자 소문자
  
  - 마침표 쓰지 않기

- **예시**
  
  ```
  docs: update readme
  feat: add user login
  fix: resolve api error
  ```
